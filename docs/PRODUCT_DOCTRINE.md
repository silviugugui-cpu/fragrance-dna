# Product Doctrine

## Purpose
Define the non-negotiable identity and mission of FragranceDNA as a Persistent Olfactory Intelligence Platform.

## Owner
Product + Architecture.

## Dependencies
None.

## Canonical Responsibility
Product-level truth used to evaluate all future features and trade-offs.

## Doctrine
1. FragranceDNA is a Persistent Olfactory Intelligence Platform.
2. Primary objective: continuously build, refine, and preserve user olfactory intelligence.
3. Recommendations are one consumer of intelligence, not the core product identity.
4. The platform optimizes for understanding the user, not only predicting the next fragrance.

## Platform Laws
1. Every feature must produce persistent intelligence, consume persistent intelligence, or both.
2. Intelligence must compound over time through replayable historical interactions.
3. Users should not repeat completed work because models improved.
4. Features that bypass persistent intelligence do not belong in FragranceDNA.

## Decision Filter
A feature is eligible only if:
1. It enriches Persistent User DNA and/or Persistent Fragrance Intelligence.
2. It preserves explainability and reproducibility.
3. It respects architecture boundaries and canonical contracts.
