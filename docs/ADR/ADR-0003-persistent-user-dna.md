# ADR-0003: Persistent User DNA

## Status
Accepted

## Context
Multiple preference models cause inconsistency and regressions.
The platform requires a single source of user intelligence.

## Decision
Persistent User DNA becomes the only canonical user intelligence contract.
All learning systems contribute through events and projections.

## Consequences
1. No competing preference models allowed.
2. Test, recommendation, collection, and feedback systems align to one profile contract.
3. Backward-compatibility adapters are used during migration.
